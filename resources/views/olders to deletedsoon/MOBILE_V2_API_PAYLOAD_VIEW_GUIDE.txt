MOBILE V2 API + PAYLOAD + VIEW GUIDE
Date: 2026-04-10
Target: React Native app clone parity from web/portal
API Base: /api/v2/mobile

================================================================================
A) CORE RULES FOR APP TEAM
================================================================================
1) Use V2 endpoints first. Keep V1 only as fallback during migration.
2) Rebuild screens from:
   - data: business entities
   - ui: section metadata and rendering hints
   - actions: next allowed operations
3) Treat payload as DB-backed canonical state.
4) Where both portal and public versions exist, keep separate app flows.

================================================================================
B) AUTH FLOWS
================================================================================
Customer:
- POST /api/v2/mobile/auth/customer/login
- POST /api/v2/mobile/auth/customer/register
- POST /api/v2/mobile/auth/customer/forgot-password
- POST /api/v2/mobile/auth/customer/confirm-reset-password
- POST /api/v2/mobile/auth/customer/logout (auth)
- GET  /api/v2/mobile/auth/customer/user (auth)

Staff:
- POST /api/v2/mobile/auth/staff/login
- GET  /api/v2/mobile/auth/staff/me (auth:sanctum)
- POST /api/v2/mobile/auth/staff/logout (auth:sanctum)

Club:
- POST /api/v2/mobile/club/login
- GET  /api/v2/mobile/club/dashboard/parity
- POST /api/v2/mobile/club/passkey/request-reset
- POST /api/v2/mobile/club/passkey/confirm-reset
